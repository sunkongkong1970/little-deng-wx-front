微信小程序需要实现以下功能，帮我细化提示词用于cursor生成页面原型设计
进行微信小程序页面原型设计，微信小程序包含以下功能：
1.用户首次登录时需要进行授权登录，获取微信小程序的openid，获取昵称和头像
2.用户在首次进入微信小程序时需要添加家庭或加入家庭，需要设计一个初始页面，页面包括一个“添加家庭”按钮，一个“加入家庭”按钮，“加入家庭”按钮前有一个输入框用于输入邀请码。
3.用户在新建家庭或者加入家庭后需要选择自己的角色，填写自己的信息，需要一个独立页面显示当前用户的昵称和头像，并且有一个下拉框可以选择自己的角色，角色包括"爸爸","妈妈"。该页面可以修改昵称和头像，修改头像的方式为点击页面上的圆形头像，下方弹出列表，列表包括“使用微信头像”，“从相册中选择”。
4.用户在新建家庭成功后跳转到小程序宝宝页面，该页面下方有四个标签，分别为“宝宝”，“应用”，“时光”，“我的”。宝宝页面的标签可以由特殊用户修改，修改后该页面标签文字会对对应的家庭成员进行展示。特殊家庭成员或用户才显示“应用”标签
5.“宝宝”页面在没有添加宝宝时显示一个矩形虚线框，中间为加号，上方文字描述为“添加宝宝”；在有宝宝时由一个矩形区域显示宝宝相片，下方矩形区域显示宝宝信息，如姓名，性别等。
6.应用页面显示多个矩形图标，每个图标可以跳转到单独功能页面；
7.时光页面，该页面复刻微信朋友圈的展示和添加朋友圈功能，增加功能：每条朋友圈底部默认展示发送时间，可展示发送位置
8.我的页面，左上角显示圆形头像，后面显示昵称。下方以列表形式展示功能：（1）修改个人资料（2）编辑宝宝简介（3）家庭权限设置（4）生成邀请码




豆包优化后的设计提示词
基于微信小程序设计规范，需生成涵盖完整用户流程与核心功能模块的高保真页面原型，要求页面风格统一（建议采用温馨亲子风，色彩以柔和暖色调为主）、交互逻辑清晰、符合微信生态用户使用习惯，使用tdesign进行设计，将生成的页面放到@prototype/ 下，生成html格式的原型设计，以iphone16为模板，将全部设计页面统一到index.html中。具体功能与页面设计细节如下：
一、核心前提与基础规范
原型需适配微信小程序标准尺寸（默认屏幕宽度 375px，支持自适应），所有页面顶部统一为微信原生导航栏（含返回按钮、页面标题），底部 tabBar 仅在 “宝宝”“应用”“时光”“我的” 页面显示。
涉及用户授权、弹窗选择等交互时，需体现弹窗触发 - 选择 - 反馈的完整流程；涉及数据状态切换（如 “无宝宝 / 有宝宝”“普通用户 / 特殊用户”）时，需分别展示不同状态下的页面样式。
二、分页面详细设计要求
1. 授权登录页
页面场景：用户首次打开小程序时触发，为进入小程序的第一个页面。
核心元素：
顶部展示小程序品牌 Logo（或亲子主题图标），下方配简短文案 “欢迎使用 XX 小程序，登录后即可管理家庭与宝宝信息”；
中间区域放置 “微信授权登录” 按钮（按钮样式符合微信规范，带微信 Logo，文字为 “微信授权登录”）；
底部标注授权说明 “登录将获取您的微信昵称、头像及 openid，用于身份识别”。
交互逻辑：点击授权按钮后，触发微信原生授权弹窗（原型中需模拟该弹窗），授权成功后跳转至 “家庭初始选择页”，授权失败则提示 “请授权以继续使用小程序” 并保留授权按钮。
2. 家庭初始选择页（首次登录后）
页面场景：用户完成授权后进入，用于选择 “添加家庭” 或 “加入家庭”。
核心元素：
页面标题为 “选择家庭方式”；
中间区域分上下两部分布局：
上部分：“加入家庭” 模块，左侧为输入框（提示文字 “请输入家庭邀请码”），右侧为 “加入家庭” 按钮（按钮与输入框水平对齐，输入框未填时按钮置灰不可点击）；
下部分：“添加家庭” 模块，仅展示 “添加家庭” 按钮（按钮尺寸与 “加入家庭” 按钮一致，独立成行，位于输入框下方 15px 处）；
底部无额外元素，保持页面简洁。
交互逻辑：点击 “添加家庭”/“加入家庭” 按钮后，均跳转至 “用户角色与信息设置页”。
3. 用户角色与信息设置页
页面场景：用户选择 “添加 / 加入家庭” 后进入，用于完善个人信息与选择角色。
核心元素：
页面标题为 “完善个人信息”；
顶部为头像设置区：展示圆形头像占位（初始为微信默认头像），下方标注 “点击更换头像”；
中间为信息编辑区：
昵称输入框（默认填充微信昵称，可修改，提示文字 “请输入您的昵称”）；
角色选择下拉框（默认显示 “请选择角色”，下拉选项仅 “爸爸”“妈妈”，选择后显示对应角色名称）；
底部为 “保存信息” 按钮（按钮宽度占页面 80%，信息未填写完整时置灰不可点击）。
交互逻辑：
点击圆形头像：弹出底部操作列表（含 “使用微信头像”“从相册中选择” 两个选项，列表带半透明背景遮罩），选择后更新头像占位图；
点击 “保存信息”：若为 “添加家庭” 则跳转至 “宝宝页面”；若为 “加入家庭” 则验证邀请码有效性后跳转至 “宝宝页面”。
4. 宝宝页面（底部 tabBar 首页）
页面场景：用户完成家庭与角色设置后进入，核心展示宝宝信息，底部含 tabBar（“宝宝”“应用”“时光”“我的”）。
核心规则：
tabBar 权限控制：仅 “特殊用户”（需标注特殊用户标识，如家庭创建者）可见 “应用” 标签，普通用户 tabBar 仅显示 “宝宝”“时光”“我的”；
tabBar 文字可修改：特殊用户可编辑 “宝宝” 页面对应 tabBar 文字（原型中需体现 “编辑 tab 文字” 入口，如右上角 “编辑” 按钮），修改后所有家庭成员看到的 tab 文字同步更新。
页面元素（分两种状态）：
无宝宝状态：
页面中间展示矩形虚线框（尺寸建议宽 200px、高 200px，虚线颜色为浅灰色），框内居中显示 “+” 号图标（图标颜色与主色调一致）；
虚线框上方显示 “添加宝宝” 文字（字体大小 16px，居中对齐）；
点击虚线框 /“添加宝宝” 文字，触发 “添加宝宝” 弹窗（模拟弹窗，含 “上传宝宝照片”“填写宝宝信息” 选项）。
有宝宝状态：
页面上半部分：矩形宝宝照片展示区（尺寸与无宝宝时虚线框一致，边框为实线浅灰色，照片居中填充，支持点击放大查看）；
页面下半部分：宝宝信息展示区（矩形区域，与照片区宽度一致，背景色为浅米色），内部按顺序展示：
宝宝姓名（字体大小 18px，加粗）；
宝宝性别（标注 “男宝 / 女宝”，带对应性别图标）；
其他基础信息（如出生日期，格式 “YYYY-MM-DD”，字体颜色为深灰色）；
信息区右侧可添加 “编辑” 按钮（图标为铅笔，点击可修改宝宝信息）。
5. 应用页面（底部 tabBar 子页，仅特殊用户可见）
页面场景：特殊用户点击 “应用” tab 后进入，展示各类功能入口。
核心元素：
页面标题为 “应用中心”；
主体为网格布局（建议 2 列，间距 15px），每个功能入口为矩形图标 + 文字组合：
图标：圆形或方形（尺寸统一，如 60px*60px），背景色为柔和色系；
文字：图标下方标注功能名称（如 “成长记录”“疫苗提醒”“身高体重” 等，文字居中，字体大小 14px）；
网格区域顶部可添加 “功能分类” 标题（如 “宝宝管理”，字体大小 16px，加粗），支持多分类布局。
交互逻辑：点击任意功能图标，跳转至对应功能的独立页面（原型中需标注跳转指向，如 “点击→成长记录页”）。
6. 时光页面（底部 tabBar 子页，复刻微信朋友圈）
页面场景：用户点击 “时光” tab 后进入，用于发布与查看家庭动态，风格与微信朋友圈一致。
核心元素（分 “发布区” 与 “动态列表区”）：
发布区：
顶部为 “发布时光” 入口：左侧显示用户头像（圆形，与 “我的” 页面头像一致），中间为输入框（提示文字 “分享宝宝的美好时光...”），右侧为 “发布” 按钮（输入框未填时置灰）；
点击输入框后，展开完整发布面板（含 “上传图片”“选择位置” 选项，布局与微信朋友圈发布页一致）。
动态列表区：
每条动态结构：用户头像 + 昵称（居左）、发布内容（文字 + 图片，图片支持 1-9 张，布局与朋友圈一致）、底部操作栏；
底部操作栏新增规则：默认显示 “发送时间”（格式 “YYYY-MM-DD HH:MM”，字体颜色浅灰色），若发布时选择了位置，则在时间右侧显示 “位置图标 + 位置名称”（如 “📍 人民公园”）；
其他交互（如点赞、评论）与微信朋友圈一致，原型中需体现对应图标。
7. 我的页面（底部 tabBar 子页）
页面场景：用户点击 “我的” tab 后进入，展示个人信息与功能列表。
核心元素：
顶部个人信息区：
左侧显示圆形头像（与 “角色设置页” 头像一致，支持点击查看大图）；
头像右侧显示昵称（字体大小 16px，加粗）+ 角色标签（如 “爸爸”，字体大小 12px，背景色为浅粉色 / 浅蓝色，带圆角）；
信息区右侧可添加 “编辑” 图标（点击跳转至 “修改个人资料页”）。
中间功能列表区（采用微信小程序原生列表样式，每项含 “图标 + 文字 + 右箭头”）：
列表项 1：“修改个人资料”（图标为 “用户编辑”）；
列表项 2：“编辑宝宝简介”（图标为 “宝宝图标”）；
列表项 3：“家庭权限设置”（图标为 “盾牌 / 设置”，仅特殊用户可见，普通用户隐藏）；
列表项 4：“生成邀请码”（图标为 “二维码”，仅特殊用户可见，普通用户隐藏）；
底部可添加 “关于我们”“帮助中心” 等辅助选项（字体颜色浅灰色）。
交互逻辑：点击列表项跳转至对应功能页（如点击 “生成邀请码”，跳转至邀请码生成页，展示二维码 + 邀请码文字，支持 “保存图片” 功能）。
三、额外设计要求
需标注页面间的跳转关系（如用箭头或备注说明 “从 A 页→B 页” 的触发条件）；
涉及 “特殊用户” 与 “普通用户”、“有宝宝” 与 “无宝宝” 等状态差异的页面，需分别生成对应状态的原型图，避免混淆；
所有按钮、输入框、下拉框等交互元素，需标注 hover 态 / 禁用态样式（如按钮禁用时背景色变浅、文字变灰）。
